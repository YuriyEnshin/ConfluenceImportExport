# История изменений

Все значимые изменения утилиты Confluence Page Exporter документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [Unreleased]

## [2.5.2] — 2026-04-24

### Изменено

- Команда `compare` теперь использует параллельный обход дерева страниц
  (`Parallel.ForEachAsync` с `--max-parallelism`) — ускорение сопоставимо
  с `download`/`upload` v2.5.0.
- Команда `compare` пропускает HTTP-запрос `/child/page` для листовых
  страниц, если сервер вернул `childTypes.page.value = false` в рамках
  основного GET. Экономия ~N запросов, где N — число листьев в дереве.

### Добавлено

- Тайминг-лог `[PROFILE] Compare completed in <Ms>ms` на уровне
  Information — тело `CompareAsync` обёрнуто в `try/finally`, лог
  пишется на всех путях выхода (включая ошибки).
- Регрессионные тесты leaf-skip и параллельной корректности
  `CollectRemotePagesAsync` (сбор 10 детей из `ConcurrentDictionary`
  без потерь под параллелизмом).

## [2.5.1] — 2026-04-24

### Добавлено

- Инфраструктурное логирование таймингов на уровне Information для команд
  `download update`, `download merge`, `upload update`, `upload merge`,
  `upload create`: после завершения операции выводится строка
  `[PROFILE] <Command> completed in <Ms>ms` для оценки длительности
  синхронизации в продуктивной нагрузке.
- HTTP-трейсинг (`HttpTimingHandler`) на уровне Debug: для каждого запроса
  к Confluence REST API фиксируется метод, URL, статус, длительность и
  размер ответа.
- Тайминги SHA-256 сравнений контента и вложений на уровне Debug
  (размер буфера + миллисекунды) в `DownloadService` и `UploadService`.

### Исправлено

- `UploadCreateAsync`: тайминг-лог `[PROFILE] UploadCreate completed`
  теперь пишется на всех путях выхода (включая ранний `return` при
  `createResult == null`) — тело обёрнуто в `try/finally`. Ранее на пути
  отказа профилировочные данные терялись.

## [2.5.0] — 2026-04-24

### Добавлено

- Параллельный обход дерева страниц и загрузка вложений в командах `download` и `upload`: на реальных пространствах (~130 страниц) сокращает время синхронизации в 3–5 раз.
- Параметр `--max-parallelism N` (по умолчанию 8) для регулировки уровня параллелизма команд `download` и `upload`. Значение `1` отключает параллелизм.

### Изменено

- Команда `download` пропускает лишние HTTP-запросы метаданных (`child/page`, `child/attachment`) для страниц-листьев: Confluence возвращает флаги `childTypes.page.value` и `childTypes.attachment.value` в рамках основного GET, и для листьев эти вызовы более не выполняются. Экономия 100+ HTTP-запросов на типичном пространстве.

## [2.4.0] — 2026-04-14

### Изменено

- Конфигурация download теперь поддерживает отдельные подсекции `Download:Update` и `Download:Merge` с наследованием общих параметров из `Download`
- Конфигурация upload поддерживает общие параметры на уровне `Upload` (например, `SourceDir`), наследуемые подкомандами `Update`, `Create`, `Merge`
- Команда `config show` отображает все подкоманды единообразно: `Download > Update`, `Download > Merge`, `Upload > Update`, `Upload > Create`, `Upload > Merge`, `Compare`

### Добавлено

- Секция `Upload > Merge` в выводе `config show`
- Параметр `DetectSource` в секции `Compare` в выводе `config show`

## [2.3.0] — 2026-04-11

### Добавлено

- Сохранение оригинального заголовка страницы Confluence в маркере `.id*`: при скачивании
  заголовок записывается в содержимое маркерного файла и используется при загрузке обратно
  на сервер, что предотвращает переименование страниц из-за санитизации имён папок.
- Автоматическое восстановление оригинальных заголовков при `upload update` и `upload merge`:
  если папка не была переименована пользователем, используется сохранённый заголовок
  (с кавычками, двоеточиями и другими символами, недопустимыми в именах файлов).
- Детекция переименования папки пользователем: если имя папки не совпадает с санитизированной
  формой сохранённого заголовка, папка считается переименованной и новое имя используется
  как заголовок страницы при загрузке на сервер.
- Серверная защита от утечки санитизированных имён: при отсутствии сохранённого заголовка
  в маркере утилита сверяет локальное имя с серверным заголовком и не выполняет
  переименование, если имя папки — это санитизированная форма серверного заголовка.

### Исправлено

- Непоследовательная замена недопустимых символов в именах папок: `SanitizeFileName` заменён
  на посимвольную замену каждого недопустимого символа на `_` вместо `Split/Join`, что
  устраняет асимметричное поведение (например, `Модуль "Провайдеры"` ранее давал
  `Модуль _Провайдеры`, теперь — `Модуль _Провайдеры_`).

## [2.2.1] — 2026-04-08

### Исправлено

- Ложные различия в `compare` для страниц, заголовок которых заканчивается точкой (`.`)
  или пробелом: Windows автоматически удаляет завершающие точки и пробелы из имён
  папок, из-за чего локальное имя не совпадало с серверным заголовком. Теперь
  `SanitizeFileName` явно обрезает завершающие точки и пробелы перед созданием папки.

## [2.2] — 2026-04-07

### Исправлено

- Вложения страниц загружались при каждом выполнении `download update` и `download merge`,
  даже если не были изменены на сервере. Добавлена двухуровневая проверка: сначала сравнение
  размера локального файла с `extensions.fileSize` из API (быстрый путь без скачивания),
  затем — при расхождении размеров — сравнение SHA-256 хешей скачанного и локального файла
  (обработка случая, когда Confluence Server перекодирует JPEG через ImageIO и отдаёт файл
  с размером, отличным от указанного в метаданных).

## [2.1] — 2026-03-26

### Исправлено

- Ложные различия при сравнении контента из-за несущественных различий в форматировании
  storage format. Перед сравнением выполняется нормализация: окончания строк (CRLF → LF),
  отступы между XML-тегами, порядок атрибутов, формат self-closing тегов (`<br/>` vs `<br />`),
  HTML-сущности (`&mdash;` vs `—`). При невозможности XML-парсинга применяется fallback
  на нормализацию окончаний строк. Затрагивает команды `compare`, `download merge`
  и `upload merge`.

## [2.0] — 2026-03-20

### Добавлено

- Команда `download update` — принудительное скачивание страниц с сервера,
  перезаписывает локальные изменения (делает локальную копию идентичной серверной).
- Команда `download merge` — умное скачивание страниц с сервера с сохранением
  локальных изменений: перезаписываются только страницы, изменённые на сервере;
  локальные правки не затираются. Конфликты (двойные правки) выявляются и
  пропускаются с предупреждением.
- Команда `upload merge` — загрузка локальных страниц на сервер с сохранением
  серверных изменений: загружаются только страницы, изменённые локально; серверные
  правки не затираются. Конфликты (двойные правки) выявляются и пропускаются
  с предупреждением.
- Ключ `--report` — при наличии после завершения `download`/`upload` выводится сводка
  страниц, требующих ручной обработки (конфликты двойных правок, пропущенные страницы).
- Определение конфликтов двойных правок: если серверная версия страницы новее маркера
  и при этом локальный файл изменён после последней синхронизации — страница
  отмечается как конфликтная и не перезаписывается ни в одну сторону.

### Изменено

- **BREAKING:** Команда `download` разделена на подкоманды `download update`
  и `download merge`. Прежний вызов `download` необходимо заменить на `download update`.
- **BREAKING:** Команда `upload update` упрощена: перемещение страниц выполняется
  всегда автоматически.
- Обновление вложений при `upload update` и `upload merge`: вместо удаления и
  повторного создания вложение обновляется через версионирование Confluence
  (новая версия файла). Неизменённые вложения пропускаются (сравнение по размеру
  файла и SHA-256 хэшу).

### Удалено

- **BREAKING:** Параметр `--overwrite-strategy` (заменён разделением `download`
  на `update`/`merge`).
- **BREAKING:** Параметр `--on-error` (в режиме `update` ошибки прерывают выполнение).
- **BREAKING:** Параметр `--move-pages` (перемещение страниц выполняется автоматически
  в `update`).

### Исправлено

- Ошибка «XSRF check failed» при загрузке вложений: добавлен обязательный заголовок
  `X-Atlassian-Token: nocheck` к запросам работы с вложениями.
- Ошибка «MethodNotAllowed» при удалении вложений: исправлен URL endpoint.

## [1.1] — 2026-03-17

### Добавлено

- Версия страницы в маркере `.id<pageId>_<version>`: при `download` и `upload update`
  маркер автоматически создаётся или обновляется, отражая актуальный id и номер версии
  страницы на сервере. Обратная совместимость со старым форматом `.id<pageId>` сохранена.
- Пропуск неизменённых страниц при `upload update`: перед отправкой на сервер сравниваются
  заголовок, контент и родительская страница — если ничего не изменилось, обновление
  пропускается, что предотвращает создание лишних версий в Confluence.
- Определение источника изменений контента по версии маркера в команде `compare`:
  если версия маркера совпадает с серверной — изменение локальное (высокая достоверность);
  если серверная версия новее — изменение на сервере (высокая достоверность).

## [1.0] — 2026-03-15

### Добавлено

- Определение источника изменений в команде `compare`: для каждого обнаруженного различия
  (переименование, перемещение, изменение контента) выводится вероятный источник — сервер
  или локальная копия — с указанием уровня достоверности (низкая / средняя / высокая).
- Флаг `--detect-source` в команде `compare`: включает анализ истории версий Confluence
  для повышения точности определения источника переименований и перемещений.
- Получение истории версий страницы через Confluence REST API
  (`GetPageVersionsAsync`, `GetPageAtVersionAsync`).
- Сервис `ChangeSourceAnalyzer` — определение источника различий по датам модификации
  и истории версий; кэширование запросов к API.
- Дата последней модификации серверной страницы (`version.when`) передаётся в снимки
  и используется при сравнении.
- Даты модификации локальных папок и файлов `index.html` передаются в снимки
  и используются при сравнении.

### Исправлено

- Команда `upload update` не обнаруживала перемещение корневой страницы (указанной
  через `--source-dir`) к другому родителю. Теперь при наличии `.id`-маркера
  в родительской директории и включённом `--move-pages` корневая страница корректно
  перемещается; при отключённом `--move-pages` — выводится ошибка.
- Команда `compare` не обнаруживала изменение родительской страницы для корневой
  страницы сравнения. Добавлена явная проверка `.id`-маркера родительской директории
  против серверного `ParentId` с анализом источника изменений.

## 2026-03-13

### Добавлено

- Команда `config show` — отображение действующей конфигурации с указанием источника
  каждого значения (`[CLI]`, `[ENV]`, `[FILE]`, `[DEFAULT]`).
- Многоуровневая конфигурация: загрузка параметров из JSON-файла (`--config`), переменных
  окружения (префикс `CONFLUENCE_EXPORTER__`) и аргументов командной строки с приоритетом
  CLI > ENV > FILE > DEFAULT.
- Переход на `Microsoft.Extensions.Hosting` с двухфазным запуском через `BootstrapParser`
  и `CommandDispatcher`.
- Секционные опции для каждой команды (`Download`, `Upload:Update`, `Upload:Create`,
  `Compare`) через `Microsoft.Extensions.Options`.
- Общие (recursive) параметры на уровне корневой команды, наследуемые всеми подкомандами.

### Изменено

- Нормализация путей: кавычки и экранированные пробелы в аргументах `--output-dir` и
  `--source-dir` обрабатываются корректно через `PathNormalizer`.

## 2026-02-27

### Добавлено

- Начальная реализация утилиты.
- Команда `download` — выгрузка страницы (или поддерева страниц) из Confluence на диск.
- Команда `upload update` — обновление существующих страниц Confluence по локальному
  содержимому.
- Команда `upload create` — создание новых страниц в Confluence по локальной структуре.
- Команда `compare` — сравнение дерева страниц Confluence с локальным снимком: обнаружение
  добавленных, удалённых, переименованных/перемещённых страниц и изменений контента.
- Локальный формат хранения: папка на страницу, `index.html`, `.id<pageId>` маркер,
  вложения как файлы.
- Сопоставление страниц по `.id<pageId>` маркерам и fallback по заголовкам
  (`--match-by-title`).
- Рекурсивная обработка дерева страниц (`--recursive`).
- Аутентификация: `--auth-type onprem` и `--auth-type cloud`.
- Подробное логирование (`--verbose`).
- Режим пробного запуска (`--dry-run`).
- Выбор страницы по `--page-id` или `--page-title`.
- Сервисная архитектура: `DownloadService`, `UploadService`, `CompareService`,
  `LocalStorageHelper`, `HttpClientConfluenceApiClient`.
