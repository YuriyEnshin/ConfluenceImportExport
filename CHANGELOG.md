# История изменений

Все значимые изменения утилиты Confluence Page Exporter документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [Unreleased]

### Добавлено

- Определение источника изменений в команде `compare`: для каждого обнаруженного различия
  (переименование, перемещение, изменение контента) выводится вероятный источник — сервер
  или локальная копия — с указанием уровня достоверности (низкая / средняя / высокая).
- Флаг `--detect-source` в команде `compare`: включает анализ истории версий Confluence
  для повышения точности определения источника переименований и перемещений.
- Получение истории версий страницы через Confluence REST API
  (`GetPageVersionsAsync`, `GetPageAtVersionAsync`).
- Сервис `ChangeSourceAnalyzer` — определение источника различий по датам модификации
  и истории версий; кэширование запросов к API.
- Дата последней модификации серверной страницы (`version.when`) передаётся в снимки
  и используется при сравнении.
- Даты модификации локальных папок и файлов `index.html` передаются в снимки
  и используются при сравнении.

## 2026-03-13

### Добавлено

- Команда `config show` — отображение действующей конфигурации с указанием источника
  каждого значения (`[CLI]`, `[ENV]`, `[FILE]`, `[DEFAULT]`).
- Многоуровневая конфигурация: загрузка параметров из JSON-файла (`--config`), переменных
  окружения (префикс `CONFLUENCE_EXPORTER__`) и аргументов командной строки с приоритетом
  CLI > ENV > FILE > DEFAULT.
- Переход на `Microsoft.Extensions.Hosting` с двухфазным запуском через `BootstrapParser`
  и `CommandDispatcher`.
- Секционные опции для каждой команды (`Download`, `Upload:Update`, `Upload:Create`,
  `Compare`) через `Microsoft.Extensions.Options`.
- Общие (recursive) параметры на уровне корневой команды, наследуемые всеми подкомандами.

### Изменено

- Нормализация путей: кавычки и экранированные пробелы в аргументах `--output-dir` и
  `--source-dir` обрабатываются корректно через `PathNormalizer`.

## 2026-03-10

### Изменено

- Обновление фреймворка тестирования: переход с xUnit 2 на xUnit v3.

## 2026-03-07

### Добавлено

- Параметр `--move-pages` в команде `upload update`: перемещение страницы под нового
  родителя, если её позиция в локальном дереве отличается от серверной.

## 2026-03-04

### Добавлено

- Обработка ошибок команд: понятные сообщения об ошибках для пользователя
  (отсутствие обязательных параметров, несуществующие директории и т.д.).
- Валидация локальных директорий перед выполнением операций (`ValidateSourceDirectory`).

### Исправлено

- Корректное определение имени папки страницы при наличии завершающего разделителя в пути.

## 2026-02-28

### Добавлено

- Проект модульных тестов `ConfluencePageExporter.Tests` (xUnit, Moq, FluentAssertions).
- Моки для API-клиента и HTTP-обработчика (`ApiClientMockFactory`, `StubHttpMessageHandler`).

## 2026-02-27

### Добавлено

- Начальная реализация утилиты.
- Команда `download` — выгрузка страницы (или поддерева страниц) из Confluence на диск.
- Команда `upload update` — обновление существующих страниц Confluence по локальному
  содержимому.
- Команда `upload create` — создание новых страниц в Confluence по локальной структуре.
- Команда `compare` — сравнение дерева страниц Confluence с локальным снимком: обнаружение
  добавленных, удалённых, переименованных/перемещённых страниц и изменений контента.
- Локальный формат хранения: папка на страницу, `index.html`, `.id<pageId>` маркер,
  вложения как файлы.
- Сопоставление страниц по `.id<pageId>` маркерам и fallback по заголовкам
  (`--match-by-title`).
- Рекурсивная обработка дерева страниц (`--recursive`).
- Аутентификация: `--auth-type onprem` и `--auth-type cloud`.
- Подробное логирование (`--verbose`).
- Режим пробного запуска (`--dry-run`).
- Выбор страницы по `--page-id` или `--page-title`.
- Сервисная архитектура: `DownloadService`, `UploadService`, `CompareService`,
  `LocalStorageHelper`, `HttpClientConfluenceApiClient`.
